このブログがどうやって生まれたのか、を残しておこうと思います。

きっかけは「ツールを作りたい」という一点でした。公開できるブログを用意しようと思ったのは、その少し後のことです。金融SEとして20年システム開発をしてきた立場から、Claude Codeという道具と向き合ってみた体験記です。

---

## きっかけは、投資シミュレーターを自作したこと

発端は[取崩しシミュレーター](/tools/montecarlo/)です。

4%ルールを自分のポートフォリオに当てはめて考えていたとき、「既存のツールでは条件を細かく設定できない」と感じました。ならば作ってしまおう、と。

仕事でも使っているClaude Codeに「こういう計算ロジックで、こういう見た目のツールが欲しい」と伝えながら、HTML/CSS/JavaScriptの単一ファイルとして実装しました。最大100万通りのシナリオを計算するモンテカルロ法のロジックも、自分で書くよりずっとスムーズに形になりました。

ツールが動くようになると、「これを公開したい、せっかくなら解説記事も書きたい」という気持ちが出てきました。それがこのブログの始まりです。

---

## 使った技術スタック

| 用途 | 採用したもの |
|---|---|
| AIアシスタント | Claude Code |
| 静的サイトジェネレーター | Hugo（PaperModテーマ） |
| ホスティング | Cloudflare Pages（無料枠） |
| ソースコード管理 | GitHub |

Hugoを選んだのは、MarkdownでHTMLを生成できる速さと、PaperModというシンプルで読みやすいテーマが気に入ったからです。Cloudflare Pagesは無料でカスタムドメインも使えるため、個人ブログの入り口として十分でした。

## 作業の流れ

### 1. コンセプト設計

まず「このブログで何を発信するか」をClaude Codeと整理しました。

ターゲット読者・4つのコンテンツ柱（高配当株の実践記録 / AI×資産管理 / 金融の仕組み解説 / 金融ニュース深掘り）・収益化の方針（AdSense + クレカアフィリエイト）——これらをドキュメントとして残しながら進めました。

### 2. Hugo + PaperModのセットアップ

HugoのインストールからPaperModテーマの適用、`config.yaml`の設定まで、コマンドの意味を確認しながら一つずつ進めました。「このオプションの意味は？」「日本語環境でのメニュー設定は？」といった質問にその場で答えてもらいながら進める作業は、リファレンスを行き来するより格段に速いと感じました。

### 3. 記事の執筆

記事もClaude Codeと並行して作りました。「こういう内容の記事を、こういう読者向けに書きたい」という方針を伝えると、構成案と下書きが出てきます。それを自分の言葉と実体験で肉付けしていく進め方が、自分には合っていました。

### 4. Cloudflare Pagesへのデプロイ

GitHubリポジトリと接続してデプロイし、カスタムドメイン（finlab-se.com）を設定しました。SSL証明書の自動発行もCloudflareが処理してくれるため、手間は少なかったです。

---

## ハマったポイント3選

順調に見えましたが、実際には何度かつまずいています。

### その1: 記事を書いたのに表示されない

最初の記事を書いてビルドしたとき、サイト上に何も表示されませんでした。

原因は**日付のタイムゾーン問題**でした。frontmatterに書いた`date: 2026-04-12`がUTCで解釈されると、日本時間では「まだ未来の記事」として扱われ、デフォルトでは非表示になります。

`hugo server --buildFuture`フラグを付けることで確認できるようになりました。本番ビルドでも同様のフラグを付けるか、日付に時刻とタイムゾーンを明示することで解決しました。

### その2: Cloudflare PagesとGitHubの認証がループする

CloudflareのダッシュボードからGitHubのOAuth連携を試みたところ、認証画面がループして先に進めなくなりました。

解決策は**Wrangler CLI（Cloudflareの公式CLIツール）を使った直接デプロイ**です。ブラウザ上の連携をあきらめ、ターミナルから`npx wrangler pages deploy`でデプロイする方法に切り替えました。CLIからの操作は素直に動き、以降はこちらを使っています。

### その3: Wranglerの認証でポートが使用中になる

Wranglerの認証コマンドを実行すると「ポート8976が使用中」というエラーが出ました。

前回の認証プロセスが残っていたためです。Windowsのタスクマネージャーで該当プロセスを強制終了し、再実行することで解決しました。

## 金融SEとして感じたこと

仕事でシステム開発をしていると、「欲しいものを要件定義して、設計して、実装する」という流れに慣れています。Claude Codeを使った個人開発は、この流れと本質的に同じでした。違いは**自分がユーザーであり、決裁者でもある**という点です。

「この機能はいらない、もっとシンプルにしたい」「この表現は読者に伝わりにくい」という判断を、誰かに通すことなく即座に反映できます。仕事の開発では当然存在する調整コストがゼロです。

また、コードを1行も書かなくてもツールが動くわけではありませんが、「アイデアから動くものが数時間で出来上がる」スピード感は、業務のシステム開発では体験しにくいものでした。

記事執筆・ツール改修・設定変更を同じ画面で並行して進められる点も、個人開発の体験として新鮮でした。
