高配当株投資を続けていると、銘柄が増えるにつれて「管理」が大きな課題になってきます。

私が高配当株を本格的に始めたのは2020年頃です。それまでのインデックス投資に加えて「日々のキャッシュフローも育てたい」という目的が明確になったことがきっかけでした。

現在のポートフォリオは、**米国高配当ETF（VYMをメイン、HDVも保有）と日本の個別高配当株の二本立て**です。日本株は約100銘柄を保有しており、これ以上は増やさない方針でセルフキャップをかけています。銘柄数が多くなるほど、利回りの確認・比率のバランス・次の買い増し候補の絞り込みといった管理作業もどんどん重くなります。

利回りはどのくらいか、1銘柄の比率が偏りすぎていないか、次に買い増すべき銘柄はどれか——こうした確認を毎回手作業でやっていると、時間がかかるうえにミスも起きやすくなります。

この記事では、私が実際に使っているポートフォリオ管理フローを公開します。MoneyForwardのデータをGoogleスプレッドシートに取り込み、Claude Codeへの自然言語指示で買い増し候補まで自動抽出する仕組みです。

## 管理フローの全体像

まず全体像から押さえます。私のポートフォリオ管理は、大きく3つのステップで構成されています。

{{< figure src="/images/highyield-portfolio-management-flow.png" alt="ポートフォリオ管理3ステップのフロー図" >}}

このフローを月1〜2回のペースで回しています。以前は手作業で1時間以上かかっていた作業が、今は30分以内で終わるようになりました。

## ステップ①：MoneyForwardで資産データを更新する

まず、MoneyForwardで全口座の残高を最新状態に更新します。

証券口座（SBI・楽天など）を連携していれば、保有株式の時価・評価損益が自動で取得されます。ただし、自動更新は数時間ごとに行われるため、確認前に手動更新ボタンを押して最新化しておくのが私の習慣です。

その後、MoneyForwardの「保有株式」画面からTSV形式（タブ区切りテキスト）でデータをコピーします。このTSVには次の情報が含まれます。

| 項目 | 内容 |
|---|---|
| 銘柄名 | 会社名 |
| 証券コード | 4桁の株式コード |
| 保有株数 | 現在の保有株数 |
| 取得単価 | 平均取得単価（円） |
| 現在値 | 最新の株価（円） |
| 時価評価額 | 現在値×保有株数 |
| 評価損益 | 含み益・含み損 |

---

## ステップ②：スプレッドシートにデータを貼付する

コピーしたTSVデータを、Googleスプレッドシートの所定の列に貼り付けます。

スプレッドシートには、MoneyForwardのデータに加えて**自分で入力・管理する列**も設けています。

| 列名 | データソース | 内容 |
|---|---|---|
| 証券コード | MoneyForward | 銘柄を識別するキー |
| 銘柄名 | MoneyForward | 会社名 |
| 保有株数 | MoneyForward | 現在の保有数 |
| 時価評価額 | MoneyForward | 現在の時価 |
| 予想配当金（1株） | 手入力 | 会社の予想配当（円） |
| 配当利回り | 自動計算 | 予想配当÷現在株価×100 |
| セクター | 手入力 | 業種分類（銀行・食品など） |
| ポートフォリオ比率 | 自動計算 | 当銘柄の時価÷総時価×100 |

**配当利回りの計算式（スプレッドシート）：**
```
= 予想配当金（1株） / 現在値 × 100
```

**ポートフォリオ比率の計算式：**
```
= 当銘柄の時価評価額 / SUM(全銘柄の時価評価額) × 100
```

予想配当金は各社のIR情報や決算短信から確認して手入力しています。決算発表後に数値を更新するのが少し手間ですが、ここだけは自動化できていない部分です。

## 買い増し候補の選定基準

スプレッドシートの準備ができたら、いよいよ買い増し候補の絞り込みです。私が判断に使っている基準は、シンプルに次の2つです。

### 基準1：配当利回りが3.8%以上

なぜ3.8%なのか。配当金には特定口座の場合、約20.315%の税金がかかります。税引後の手取り利回りを3%以上確保するには、税引前で3.78%以上が必要です。端数を丸めて3.8%を基準にしています。

```
税引後利回り = 税引前利回り × (1 - 0.20315)
3.8% × 0.79685 ≒ 3.03%  ← 手取り3%以上を確保
```

NISAの成長投資枠で保有している銘柄は課税されないため、この計算は特定口座の銘柄向けです。NISA枠の銘柄はより低い利回りでも許容しています。

### 基準2：ポートフォリオ比率が1%未満

1銘柄の比率がポートフォリオの1%を超えている場合、その銘柄はそれ以上買い増しません。1銘柄への集中リスクを避けるためです。

50〜100銘柄に分散する場合、均等配分なら1銘柄あたり1〜2%になります。1%未満という基準はやや厳しめですが、保有銘柄数が増えるほど分散効果が高まり、個別銘柄のリスク（減配・倒産）の影響を小さく抑えられると考えています。

## ステップ③：Claude Codeで買い増し候補を自動抽出する

スプレッドシートのデータが最新化できたら、Claude Codeに指示を出して買い増し候補を抽出します。

**実際にClaude Codeへ出している指示の例：**

> 「スプレッドシートの高配当株管理シートを確認して、配当利回りが3.8%以上かつポートフォリオ比率が1%未満の銘柄を一覧にしてください。セクター別にまとめてください。」

この一言だけで、条件を満たす銘柄がセクター別に整理されたリストになって返ってきます。自分でフィルタをかけてソートして……という手作業が不要になりました。

**抽出結果のイメージ：**

| セクター | 銘柄名 | 利回り | 比率 |
|---|---|---|---|
| 銀行 | ○○銀行 | 4.2% | 0.8% |
| 食品 | ○○食品 | 3.9% | 0.7% |
| 通信 | ○○通信 | 4.5% | 0.5% |

このリストをもとに、資金状況と照らし合わせながら実際に何を買い増すかを判断します。Claude Codeはあくまで「絞り込みのサポート」であり、最終判断は自分で行うのが原則です。

## スプレッドシートを自分で作る場合のポイント

「同じような仕組みを自分でも作ってみたい」という方向けに、最低限必要な列を挙げておきます。

1. **銘柄コード**（データのキーになる）
2. **時価評価額**（MoneyForwardから取得）
3. **予想配当金（1株）**（手入力）
4. **配当利回り**（自動計算）
5. **ポートフォリオ比率**（自動計算）

セクター分類は手入力になりますが、一度入力してしまえばその後の更新は不要です。

スプレッドシートの構成はシンプルであるほど、後からClaude Codeに指示を出したときに正確に処理されやすいと感じています。凝った作りにするより、シンプルで一貫性のある構造を心がけるのがおすすめです。

## 管理フローで気をつけていること

このフローを長く回すなかで、特に意識しているポイントを5つ紹介します。

### 配当データの鮮度を保つ

予想配当金は決算発表後に変わることがあります。減配・増配の情報を見逃すと利回りの計算がずれてしまうため、決算シーズン（3月・9月前後）には各銘柄のIR情報を確認するようにしています。

### 買い増し候補＝必ず買うわけではない

条件を満たす候補が出ても、手元の資金状況や市場全体の状況によっては買わないこともあります。このフローはあくまで「候補の絞り込み」であって、「自動売買」ではありません。

### 銘柄数に上限を設ける

私は日本の個別株を約100銘柄保有しています。管理のしやすさと分散効果のバランスを考えると、100銘柄を超えたあたりから管理コストが上がる割に分散効果の追加分は薄くなる、と感じています。そのため、新しく買い増すときは既存銘柄への追加か入れ替えで対応し、銘柄数自体は増やさないようにしています。

### セクター偏りを定期的にチェックする

個別の銘柄比率だけでなく、セクター全体の比率も確認します。たとえば「銀行株ばかり10銘柄集まって合計で15%になっている」という状況は、個別比率が1%未満でもセクター集中リスクを抱えていることになります。

### 米国ETFは利回りを見てタイミングを判断する

私のポートフォリオの目標は、日本の高配当個別株と米国高配当ETF（VYM・HDV）からの配当収入を**半々に近づけること**です。現状はバランスが偏っており、米国ETF側の比率がまだ低い状態です。

とはいえ、ここ2年ほどは米国高配当ETFの配当利回りが過去の平均と比べて低く、割高に感じているため、積極的に投資額を増やせていません。「利回りが低いときは割高」という判断基準を持ち、水準が戻るタイミングを待つのもひとつの選択肢です。日本株の個別分散と米国ETFの成長性・為替分散を組み合わせることで、配当の安定と資産成長の両立を目指しています。

## まとめ

私のポートフォリオ管理フローをまとめると次のとおりです。

1. **MoneyForwardで最新データを取得**（月1〜2回）
2. **スプレッドシートにTSVデータを貼付**し、配当利回り・比率を自動計算
3. **Claude Codeへの自然言語指示**で買い増し候補を自動抽出

「利回り3.8%以上かつ比率1%未満」という2つの数値基準があることで、感情に左右されず機械的に候補を絞り込めるのが気に入っています。
