高配当株を20〜50銘柄と持ち始めると、ふと気になることがあります。

「結局、今年いくら配当が入るんだっけ？」「どの月に入金が偏っているんだろう？」

証券会社のアプリでは銘柄ごとの配当履歴は見られますが、**月別の合計キャッシュフロー**を横断で見るのは意外と面倒です。Excelで組めば作れますが、銘柄の入れ替えのたびに数式を直すのは地味に重い作業です。

この記事では、Claude Code（Anthropic社のAIコーディング支援ツール）に**銘柄リスト・株数・配当月の3つを渡すだけ**で、年間の配当金カレンダーを自動生成する手順を紹介します。プログラミング未経験でも、コピペで再現できる内容です。

## この記事でわかること

- 配当金カレンダーが「銘柄選び」ではなく「人生設計」に効く理由
- Claude Codeに渡すべき3つの情報（銘柄・株数・配当月）
- そのままコピペで使えるプロンプト例と、実際の出力イメージ
- スプレッドシートやMoneyForwardとの連携アイデア
- AIに任せてよい部分と、自分で判断すべき部分の切り分け

---

## なぜ配当金カレンダーが必要なのか

高配当株投資を続けていくと、保有銘柄は自然と増えていきます。日本株だけでも20銘柄、米国ETFも含めれば50銘柄近くになる人は珍しくありません。

このとき、**年間の合計配当額**は把握していても、**月別にいくら入るか**まで見ている人は意外と少ないはずです。

### 月別の偏りは「生活の安心感」に直結する

仮に年間配当が120万円あっても、そのうち70万円が6月と12月の2か月に集中していたら、残りの10か月は毎月5万円の入金しかありません。これはこれで「ボーナス型キャッシュフロー」として悪くないのですが、**毎月の生活費を配当でカバーしたい人**にとっては偏りすぎです。

一方、米国ETFを組み合わせると3・6・9・12月と四半期ごとに配当が入るため、月別の谷が浅くなります。

つまり、配当金カレンダーを作ると次のような判断ができるようになります。

| 見えるもの | できる判断 |
|---|---|
| 月別の入金額 | 生活費・固定費との対応づけ |
| 月別の偏り | 次の買い増し候補の選び方（谷の月を埋める銘柄を優先） |
| 年間合計 | 目標配当額までの到達度 |

これは「どの銘柄が上がるか」という商品選びの話ではなく、**自分のキャッシュフローをどう設計するか**という人生設計の話です。投資の軸を「商品選び」から「設計」に移すと、日々の値動きに振り回されにくくなります。

### 手作業で作ると続かない

私自身、最初はExcelで配当カレンダーを手組みしました。しかし、銘柄を1つ入れ替えるたびに、株数・配当月・配当額を書き換える必要があります。増配があれば更新が必要ですし、NISA枠の使い方を見直すたびに作り直しです。

「作ったけど続かない」——これが個人投資家あるあるです。ここでClaude Codeの出番になります。

## Claude Codeに渡すべき3つの情報

Claude Codeに配当金カレンダーを作らせるために必要な情報は、たった3つです。

1. **銘柄名（またはティッカー）**
2. **保有株数**
3. **配当月と1株あたり配当額（年間）**

この3つがあれば、あとは「月別に集計してください」と頼むだけで表が出てきます。

### 入力データの例（CSV形式）

Claude Codeに渡すときは、CSV形式でまとめておくと扱いやすくなります。以下はサンプルです（銘柄・金額は例示であり、買い推奨ではありません）。

```csv
銘柄,株数,1株配当(円),配当月1,配当月2
三菱商事,100,100,3,9
KDDI,100,145,3,9
三井住友FG,100,330,3,9
オリックス,200,98.6,3,9
東京海上HD,100,170,3,9
NTT,1000,5.2,3,9
VYM,20,470,3/6/9/12,
```

ポイントは以下の3つです。

- **株数は「今日時点の保有数」**を書く（将来予定ではなく現状）
- **1株配当は「年間合計」**か「1回あたり×回数」か、どちらかに統一する（混ざるとバグのもと）
- **配当月が複数ある場合は区切り文字を決めておく**（スラッシュなど）

このCSVを1つ用意するだけで、以降のプロンプトはすべてここを起点に動きます。

## 実際のプロンプト例と出力結果

Claude Codeに渡すプロンプトは、次のようなシンプルなもので十分です。コピペで使える形にしています。

### プロンプト例

````text
添付のCSV（holdings.csv）は私の保有している高配当株リストです。
以下を実行してください。

1. 各銘柄の「年間配当額」を計算する（株数 × 1株配当）
2. 配当月の列を見て、各月にいくら入るかを振り分ける
   - 配当月が複数ある銘柄は、年間配当額を回数で等分する
   - VYMのように "3/6/9/12" と書かれている場合は4回均等と解釈する
3. 月別の合計配当額を出す
4. 結果を以下の2つの表にまとめる
   - 表A：銘柄ごとの月別配当マトリクス
   - 表B：月別合計のサマリー（12か月分）
5. 表Bのうち、年間合計に対する各月の割合(%)も表示する

前提：税引前・円建てで計算してよい。米国株は1ドル=150円で円換算する。
````

プロンプトのコツは次の3点です。

- **前提条件を明記する**（税引前か後か、為替レート、四捨五入の単位など）
- **出力形式を具体的に指定する**（「表で」「CSVで」「Markdownで」）
- **あいまいな記法の解釈ルールを先に渡す**（"3/6/9/12" をどう読むか、など）

この3点を押さえると、出力のブレがぐっと小さくなります。

### 出力イメージ（表B：月別サマリー）

Claude Codeからは、こういった表が返ってきます（数字は架空のサンプル）。

| 月 | 配当合計(円) | 年間比率 |
|---|---|---|
| 1月 | 14,100 | 1.5% |
| 2月 | 14,100 | 1.5% |
| 3月 | 186,520 | 19.4% |
| 4月 | 14,100 | 1.5% |
| 5月 | 14,100 | 1.5% |
| 6月 | 215,620 | 22.5% |
| 7月 | 14,100 | 1.5% |
| 8月 | 14,100 | 1.5% |
| 9月 | 186,520 | 19.4% |
| 10月 | 14,100 | 1.5% |
| 11月 | 14,100 | 1.5% |
| 12月 | 257,220 | 26.8% |
| **合計** | **958,680** | **100.0%** |

ここまで来ると、「6月と12月に偏りすぎているから、次は2月・8月配当の銘柄で谷を埋めよう」といった**次のアクション**が自然に見えてきます。

### 可視化まで頼めば、さらに読みやすい

Claude Codeは表の出力だけでなく、グラフを描くHTMLやPythonコードも生成できます。たとえば次のように追加で依頼すると、棒グラフが手に入ります。

```text
上記の月別サマリーを、棒グラフで可視化するHTMLファイルを作ってください。
Chart.jsを使い、1枚のHTMLで完結するようにしてください。
```

ファイルを開けば、そのまま棒グラフとして月別の偏りが一目で確認できます。コードを1行も書かずに可視化まで到達できるのが、AIコーディング支援ツールの強みです。

## MoneyForward・スプレッドシートへの連携アイデア

配当金カレンダーは、単体で作って終わりではなく、既存の資産管理フローに組み込むとさらに便利になります。いくつか実用的な連携パターンを紹介します。

### パターン1：MoneyForwardの保有銘柄CSVを起点にする

MoneyForward Meには、連携した証券口座の保有銘柄一覧を画面からコピーできる機能があります（有料プランの機能を含む場合あり）。コピー＆ペーストしたテキストをClaude Codeに渡せば、**保有銘柄リストを自分で打ち直す手間がなくなる**のが大きいです。

実際のフローはこうなります。

1. MoneyForwardから保有銘柄リストをコピー
2. Claude Codeに貼り付けて「これを配当カレンダー用のCSVに整形して」と依頼
3. 配当月と1株配当の情報を付け足して、カレンダー生成に進む

### パターン2：スプレッドシートの定期更新に組み込む

私は配当関連の管理もGoogleスプレッドシートで行っています。Claude Codeには、スプレッドシートに貼り付けるためのTSV（タブ区切り）形式で出力してもらうのがおすすめです。

```text
上記の月別サマリーを、Googleスプレッドシートに貼り付けやすいTSV形式で出力してください。
```

こうしておけば、結果をコピーしてシートに貼るだけで更新完了です。Apps Scriptを書く必要はありません。

### パターン3：差分チェックに使う

半年に1回、保有銘柄を少し入れ替えたあとに「前回のカレンダーと比較して、どの月の配当がどれだけ増減したか」を出してもらうのも便利です。銘柄入替の影響を**キャッシュフロー視点で評価**できるので、値上がり益だけでは見えない効果が見えてきます。

## AIに任せる部分と、自分で判断すべき部分の切り分け

ここが一番大事なところです。

Claude Codeは「集計・整形・可視化」は極めて得意ですが、**投資判断そのもの**は人間がやるべき仕事です。具体的な線引きは次の通りです。

### AIに任せてよいこと

- 株数 × 1株配当の計算
- 配当月ごとの集計・振り分け
- 表やグラフの形での可視化
- 異なるフォーマット間の変換（CSV ↔ TSV ↔ Markdown）
- 前回との差分抽出

これらはすべて「機械的に答えが一つに決まる作業」です。人間がやってもAIがやっても結果は同じで、AIのほうが早く・ミスなくこなせます。

### 自分で判断すべきこと

- どの銘柄をポートフォリオに入れるか
- 各銘柄の割合をどうするか
- 減配リスク・業績の評価
- NISA枠・特定口座の使い分け
- そもそも、いくらの配当を目標にするか

これらは**自分の人生設計に直結する判断**です。「配当月が空いているから」という理由だけで銘柄を選ぶと、配当利回りは高くても業績が怪しい銘柄を掴んでしまうことがあります。AIの出力は「材料の一つ」として扱い、最終判断は自分で下すのが原則です。

リベシティ（両学長コミュニティ）でもよく言われる「**小さく始めて、自分の頭で考える**」というスタンスは、AI活用においても変わりません。AIは作業を肩代わりしてくれますが、判断まで肩代わりしてもらうと、それは自分の投資ではなくなってしまいます。

## まとめ：まずは1枚のCSVから始めてみる

配当金カレンダーは、年間のキャッシュフローを「点」ではなく「流れ」として見るためのツールです。そして、Claude Codeを使えばプログラミング経験ゼロでも作れます。

今日からできるステップは次の3つです。

1. 保有銘柄・株数・配当月を並べたCSVを1枚だけ作る
2. Claude Codeに本記事のプロンプトをコピペで渡す
3. 出てきた月別サマリーを見て、**どの月に谷があるか**だけ確認する

ここまで10〜20分あれば終わります。大事なのは完璧な表を作ることではなく、**「自分の年間キャッシュフローの形」を自分で把握すること**です。

投資は商品選びではなく、人生設計です。配当金カレンダーはその設計図の一部になります。次の買い増し候補を考えるとき、きっと新しい視点をくれるはずです。
